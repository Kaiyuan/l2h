# l2h 项目说明
## 软件大致说明
服务器A部署网页，服务器B运行服务端并且设置指定端口，用户访问服务器A的网页后访客通过 WebRTC 连接到服务器B的服务端，从而实现网页访问服务器B运行的服务端设置端口的网页的功能。

## 服务器 A 端说明
服务器A部署的网页有一个后台功能，后台界面使用 primevue V4作为网页 UI，使用 SQLite保存相关设置。
服务部署后首次访问后端时候需要设置后台路径、用户名、密码、邮件地址等属性，设置后台路径后只有访问设置的路径才可以访问服务后台。可以设置 URL 路径和访问权限，从而实现对服务器B的服务端进行访问控制。可以运行无需密码就能访问，例如 `http://serverA/pathB` 中 `pathB` 对应服务器B的服务端的某个端口，也可以设置需要密码才能访问，例如 `http://serverA/securePathB` 需要输入密码才能访问服务器B的服务端的某个端口。
后台页面可以生成 Api key，服务器B的服务端需要使用 Api key 来验证访问权限。

网页使用命令 l2h-s --help 可以查看所有命令选项。l2h-s

## 服务器 B 端说明
服务器B 运行的服务端通过命令运行，编译后的程序为 l2h-s，并且能通过命令将访问路径绑定到指定端口，例如将 `pathB` 绑定到服务器B的 8080 端口，`securePathB` 绑定到服务器B的 9090 端口。

后端运行后默认开启一个服务端管理页面，管理页面需要在首次安装程序时候使用命令设置的用户名和密码登录以及管理页面的端口。默认端口使用55055，登录后可以查看当前所有绑定的路径和端口信息，可以删除某个路径的绑定，可以添加新的路径绑定。

程序使用命令 l2h-c --help 可以查看所有命令选项。显示当期端口管理页面账号密码信息为 l2h-c --show-admin-info 命令。-l 为显示当前绑定的路径和端口信息，每行都有对应编号，-a 为添加新的路径绑定，格式为 path:passwork, passwork 可以为空，path 属性禁止使用敏感单词，-d 为删除某个路径绑定。l2h-c -s server.com:apikey 为设置服务器A的地址和 Api key 信息，从而实现访问服务器A的网页功能。
